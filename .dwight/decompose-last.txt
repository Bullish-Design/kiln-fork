**The Root Cause:**

The layout uses `<body class="flex">` with three children: `<nav>` (left sidebar), `<main class="flex-1">`, and `<aside>` (right sidebar). The sidebars are `xl:relative` on desktop.

When sidebars are collapsed, CSS uses `transform: translateX(-100%)` / `translateX(100%)` to hide them. But `transform` does NOT remove them from document flow — they still occupy their `w-72` (288px) space in the flex layout. The sidebars just visually move off-screen.

When HTMX swaps the `<main>` (via `outerHTML`), the flex layout recalculates. Since the collapsed sidebars still take up 288px each but are visually off-screen, the `<main>` element gets squeezed and positioned incorrectly. The flex container distributes space accounting for the invisible sidebars.

The fix needs to actually remove collapsed sidebars from the flex flow on desktop — using `width: 0` and `overflow: hidden` (or `min-width: 0; flex: 0`) alongside the transform, so they don't occupy space.
